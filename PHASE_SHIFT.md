# Phase Shift

**Status:** Human OS v0.1.0-alpha is fully operational.

---

## 1. Tactical Guidelines (Next 7 Days)
To ensure system maturity and stabilize the platform for internal users, our development focus shifts exclusively from infrastructure generation to product refinement.

### High Priority
*   **Bug Fixes:** Clear the queue of identified client-side and server-side errors.
*   **Performance Improvements:** Optimize vector memory search latencies and background queue processing.
*   **UX Improvements:** Enhance chat responsiveness, onboarding questions completion, and error states handling in the UI.
*   **Crash Fixes:** Resolve any server-side process crashes or client-side runtime exceptions.

### Low Priority (Deprioritized)
*   New system architectures
*   Major database refactors
*   New storage layers or databases
*   New libraries, frameworks, or dependencies

---

## 2. Feature Filtering Framework
Every proposed new feature during this period must answer the following three questions before implementation is approved:
1.  **Which user pain does it solve?** (What barrier is preventing the user from using Nova daily?)
2.  **How often will it be used?** (Will this feature be engaged multiple times per day, or is it a fringe utility?)
3.  **Does it increase retention?** (Will it make the user return tomorrow, in 7 days, and in 30 days?)

---

## 3. Product Shift Goal
Transition the development cycle from purely "building software" to **"building a product people love."**
