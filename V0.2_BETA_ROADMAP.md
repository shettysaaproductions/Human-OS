# v0.2.0-beta Roadmap

**Goal:** Release Human OS v0.2.0-beta to 25–50 active internal/external users with advanced companion modules, multi-brain intelligence routing, and telemetry tracking.

---

## 1. Feature Specifications

### 🎤 Voice Mode (Conversational Realism)
*   **Push-to-Talk:** Manual voice controls for instant speech input/output.
*   **Continuous Conversation Mode:** Full-duplex conversational sessions without requiring manual taps (VAD-enabled trigger).
*   **Voice Memory Summaries:** Voice-synthesized audio notes recapping Nova's thoughts, dream logs, or emotional status.

### 🧠 Reflection Engine (Episodic Growth)
*   **Daily Summaries:** Background cron workers that consolidate the day's chats into key events.
*   **Weekly Summaries:** Macro-level growth trends, themes, and significant companion breakthroughs.
*   **Memory Consolidation:** Automatic migration of short-term (`working_memory`) facts into long-term semantic embeddings during off-peak hours.

### 🤝 Relationship Engine (Organic Trust)
*   **Relationship Levels:** Progressive stages of intimacy and trust (e.g. Stranger -> Acquaintance -> Companion).
*   **Milestone Tracking:** Logging shared memories, achievements, and structural conversation milestones.
*   **Emotional Context:** Fine-grained adjustments to response tone based on current profile mood and sentiment history.

### 🔔 Proactive Notifications (Meaningful Engagement)
*   **Gentle Reminders:** Non-intrusive re-engagement based on active tasks.
*   **Follow-Ups:** Automatic inquiries regarding personal goals or important events mentioned earlier.
*   **Milestone Celebrations:** Spontaneous acknowledgments of shared milestones or anniversaries.

### 🖥️ Admin Dashboard UI (Operator Center)
*   **Model Management:** Graphical interface to priority-order active LLM endpoints.
*   **API Key Rotation:** Instant, non-disruptive provider key configuration.
*   **Metrics Dashboard:** Visual display of active users, latency spikes, errors, and system costs.

### 🕸️ Multi-Brain Architecture (Optimal Routing & Failover)
*   **Router-Based Selection:** `ModelRouterService` dynamically chooses the most appropriate model based on query complexity.
*   **Cheaper Models:** Fast, lightweight models handle simple tasks, onboarding forms, and entity extractions.
*   **Premium Models:** Rich reasoning models process complex cognitive states, reflection generation, and high-empathy chat replies.
*   **Automatic Failover:** Automatic redirection of requests to secondary/backup providers upon rate-limits or timeouts.

---

## 2. Release Constraints & Rules
> [!WARNING]
> **No Major Refactors:** Avoid restructuring core backend logic or databases unless strictly required to implement a roadmap item. Leverage the adapter/service layer architecture to plug in new engines natively.
